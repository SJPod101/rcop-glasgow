Live Dashboard Backend

Turning the Glasgow Branch dashboard from a local page into a shared hub for questions, requests, answers and uploaded assets.

Dashboard Current Supabase Access Next Steps
Current position

Live link exists

The dashboard is published on GitHub Pages and can be shared now as a working reference.

Forms are local

Questions, answers and requests currently save in the browser using localStorage. That means they are useful on one device, but not yet shared across the committee.

WhatsApp is manual

The dashboard can open WhatsApp with pre-written text, but WhatsApp will not allow a normal webpage to silently post into the group.

Proposed Supabase structure
Action requests

Website updates, email nudges, asset requests, MedTech requests and decisions added from the dashboard.

Open questions and answers

Questions stay visible; committee members can answer them; answers can be copied or shared manually into WhatsApp.

Asset submissions

Back to Basics, POM-S, regular emails and AGM campaign uploads can be logged against the correct area.

Draft SQL prepared

The proposed database tables are in supabase/rcop-dashboard-schema.sql. They have not been applied yet.

Access decision needed

Option A: low friction

Anyone with the link can read and submit. This is easiest for volunteers, but it is not private.

Option B: team login

Committee members sign in with Supabase Auth/magic links. Better control, but slightly more admin.

Option C: private admin

Public pages collect submissions, but only Sarah-Jane/Michael can see the working board. Best for sensitive content.

Security note from Supabase

The connected Supabase project already contains some existing public-schema tables with Row Level Security disabled, including integration/token-looking tables. I have not changed those. Before the dashboard is wired in, those existing tables should be reviewed carefully so we do not expose anything sensitive.

Next build step
Choose access model

Decide whether the shared dashboard should be public-with-link, committee login, or private admin view.

Apply RCoP-only tables

Once confirmed, apply only the rcop_ tables and policies. Do not mix this with existing MedTech/GHL tables.

Replace localStorage

Update the dashboard forms so new requests, questions, answers and file notes save into Supabase and appear for everyone.

Add uploads

Create a storage bucket for asset uploads, grouped by Back to Basics, POM-S, regular emails, AGM and conference.
